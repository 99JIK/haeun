# Algorithm Study

코딩 테스트 대비 알고리즘 문제 풀이 및 정리

## 개요

| 항목 | 내용 |
|------|------|
| **기간** | 2025.09 ~ 2026.01 |
| **목표** | 코딩 테스트 대비 |
| **언어** | Python, C++, Java |
| **풀이 수** | 약 150문제 |
| **상태** | 🟢 완료 |

## 학습 목표

취업 준비를 위한 코딩 테스트 역량 강화. 백준과 프로그래머스를 중심으로 유형별 문제를 풀고, 풀이 패턴을 정리했다.

## 학습 로드맵

### Phase 1: 기초 자료구조 (9~10월)
- 배열, 문자열 처리
- 스택, 큐, 덱
- 해시맵, 집합
- 우선순위 큐 (힙)

### Phase 2: 탐색과 정렬 (10~11월)
- 이분 탐색, 파라메트릭 서치
- BFS/DFS 그래프 탐색
- 위상 정렬
- 정렬 알고리즘 비교

### Phase 3: 고급 알고리즘 (11~12월)
- 다이나믹 프로그래밍 (1차원, 2차원, 배낭)
- 그리디 알고리즘
- 분할 정복
- 최단 경로 (다익스트라, 플로이드)

### Phase 4: 실전 연습 (12~1월)
- 프로그래머스 Lv.2~3 집중 풀이
- 시간 제한 모의고사
- 자주 틀리는 유형 복습

## 언어별 활용

| 언어 | 용도 | 비율 |
|------|------|------|
| Python | 빠른 구현, 문자열 처리, 수학 | 60% |
| C++ | 시간 제한 빡빡한 문제, STL 활용 | 25% |
| Java | 객체지향 연습, 기업 코테 대비 | 15% |

## 핵심 정리 노트

### DP 접근법
1. 문제에서 **최적 부분 구조**와 **중복 부분 문제**가 있는지 확인
2. 상태 정의: `dp[i]`가 무엇을 의미하는지 명확히
3. 점화식 도출
4. 초기값 설정
5. Bottom-up 또는 Top-down 구현

### 그래프 탐색 선택 기준
- **BFS**: 최단 거리, 레벨 단위 탐색
- **DFS**: 경로 탐색, 백트래킹, 연결 요소

## 배운 점

- 문제를 보고 어떤 알고리즘을 적용할지 판단하는 능력이 생겼다
- 시간/공간 복잡도를 고려한 풀이 선택의 중요성
- Python의 간결함 vs C++의 성능 트레이드오프 체감
- 꾸준히 하는 게 제일 중요하다는 걸 체감
